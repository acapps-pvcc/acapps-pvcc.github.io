Briefly describe or define:
  1. information security: This means protecting data from those who do not have permission to access it. Information security could cover information locked in your filing cabinet, stuffed somewhere in your purse or wallet, or lying around on your desk.
  2. authentication: This refers to a process of verifying that you really are the person who has the right to access a particular device, whether it is your local machine or the remote server.
  3. authorization: Is the process that governs what an authenticated user is allowed to do. Once you are authenticated, there might still be operations that you are not permitted to carry out.
  4. dual authentication: A secure process in which a legitimate user verifies their identity using two forms of authentication. It typically prompts the user to enter his or her user ID and password. Each user has a small device or perhaps a smartphone app that then generates a random sequence of digits.
  5. social engineering: It is the process of using people to get the information you want. If Ravi wants Alice’s password, he might just ask her! In a business setting, he might get a chance to snoop around her office and find the yellow sticky note containing her password attached to her monitor or stuck beneath her keyboard.
  6. virus: This defines a computer program that, like a biological virus, infects a host computer and then spreads. It embeds itself within another program or file. When that program or file is activated, the virus copies itself and attacks other files on the system.
  7. worm: It is a self-replicating piece of software that can travel from node to node without any human intervention, and is very similar to a virus, but it can send copies of itself to other nodes on a computer network without having to be carried by an infected host file. It is a self-replicating piece of software that can travel from node to node without any human intervention.
  8. Trojan horse: A computer program that does some legitimate computational task but also, unbeknownst to the user, contains code to per form the same kinds of malicious attacks as viruses and worms—corrupt or delete files, slow down the computer, and the like.
  9. ransomware: This is a malware software that encrypts files on a computer with a strong encryption algorithm, only the author of the malware, who holds the decryption key, can unlock the files. Typically, The user presented with a “ransom note” demanding payment in order to release the files. Also, the attacker might warned that without payment, the files may be posted online and publicly released.
  10. phishing: Phishing is a practice used to illegally obtain sensitive information such as credit card numbers, account numbers, and passwords. An email is sent claiming to be from a legitimate organization such as a bank, credit card company, online retailer, government agency, or social media site asking the recipient to click a link to update or verify his or her account information.
  11. White Hat: This refers to a type of ethical hacker and the security experts who try to keep us safe in cyberspace.
  12. Black Hat: This means the bad guys (or gals!) who try to ferret out computer system weaknesses for financial gain, control, or outright destruction.
  13. encryption/decryption: Encryption is the process of using an algorithm to convert information into a representation that cannot be understood or utilized by anyone. Its purpose is to make information meaningless even if someone does manage to steal it. Decryption is the reverse of encryption. This using an algorithm that converts the ciphertext back into plain text, and decoded back to readable original text.
  14. cryptography: It is the science of “secret writing.” A message (plaintext) is encoded (encrypted) before it is sent, for the purpose of keeping its content secret if it is intercepted by the wrong parties.
  15. symmetric encryption algorithm: A encryption method that requires the use of a secret key known to both the sender and the receiver. The sender encrypts the plaintext using the key. The receiver, knowing the key, is easily able to reverse the process and decrypt the message.
  16. Caesar cipher: A encryption technique also called a shift cipher, it involves shifting each character in the message to another character some fixed distance farther along in the alphabet.
  17. block cipher: This is a group or block of plaintext letters gets encoded into a block of ciphertext, but not by substituting one character at a time for each letter. Each plaintext character in the block contributes to more than one ciphertext character, and each ciphertext character is the result of more than one plaintext letter.
  18. Data Encryption Standard (DES): This is a symmetric block cipher developed by IBM in the 1970s for the U.S. National Bureau of Standards. It was designed to protect electronic information and processes data in 64-bit blocks using a 56-bit key for both encryption and decryption.
  19. public key system: It is an asymmetric encryption method that uses a pair of keys; an encryption key, which is shared openly so anyone can send secure messages to a particular receiver, but the decryption key cannot be derived from it and is known only by the receiver.
  20. embedded computers: Are computational devices such as chips, processors, or computers that are embedded within another system, they are small devices that is hidden by user. Unlike a general-purpose computer that can be programmed to do many different things, embedded computers usually perform just one or two tasks as part of the system in which they occur.
Briefly describe how a text file can be hidden in a steganographic image:
 A text file can be hidden in an image file by modifying the least significant bits of the image's pixel data to encode the binary representation of the text. For example, the ASCII code for "A" is 01000001. The least significant bits of first 8 bytes of the image file would be set to 0, 1, 0, 0, 0, 0, 0, and 1. These changes are make the image appear unaltered to the naked eye.
List six ways to protect yourself from most cyber attacks
  1. Be sure your computer has up-to-date antivirus software from a repu-table company. Such software can detect worms, viruses, and Trojan horses by the distinctive “signatures” those programs carry. It cleans your machine of infected files.
  2. Be sure your computer has up-to-date firewall software from a reputa-ble company. Firewall software guards the access points to your computer, blocking communications to or from sites you don’t permit and preventing certain operations from being initiated across the Internet.
  3. Be sure your computer has up-to-date antispyware from a reputable company. Antispyware routinely scans your computer for any “spyware” programs that may have infected your machine—programs that capture information on what websites you have visited and what passwords and credit card numbers you have used.
  4. Set your browser to use a pop-up blocker. A pop-up is a small window that “pops up” on the webpage your browser is currently displaying. Most pop-ups are simply annoying advertising, but some contain links, perhaps disguised as “Close” buttons, that download a virus to your machine.
  5. Always install the latest security patches or updates to your operating system; better yet, set your machine to install automatic updates.
  6. Don’t send personal or financial information in response to any email(“My wealthy Nigerian uncle left me a fortune that I am willing to share with you but I need your account number . . . ”).
