Von Neumann computer architecture, 4 major subsystems:
  1. memory
  2. input/output
  3. arithmetic/logic unit (ALU)
  4. central processing unit(CPU)
Von Neumann computer architecture, 3 characteristics:
  1. The ALU and the control unit are often bundled together in what is called the central processing unit or CPU.
  2. The stored program concept, in which the instructions to be executed by the computer are represented as binary values and stored in memory.
  3. The sequential execution of instructions, in which one instruction at a time is fetched from memory and passed to the control unit, where it is decoded and executed.
Computer Architecture Terms
  1. RAM: This defines Computer memory uses an access technique called random access memory. This is a type of volatile memory that causes the information to disappear when the power is turned off. Without some type of long-term, nonvolatile memory, information could not be saved between shutdowns of the machine.
  2. cache: A temporary high-speed memory unit that keeps a copy of memory cells are likely to be used again in the near future.
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. It will access that same instruction or piece of data in the very near future.
    2. It will soon access the instructions or data that are located near that piece of data, where “near” means an address whose numerical value is close to this one.
  4. ROM: Read-only memory (ROM) is a type of non-volatile memory that stores data permanently. This information has been prerecorded during manufacture, which cannot be modified or removed, only fetched. ROM is used to hold important system instructions and data in a place where a user cannot accidentally or intentionally overwrite them.
  5. memory cell size: The memory unit is made up of cells that contain a fixed number of binary digits, typically 8-bits per cell or 1 byte.
  6. memory cell address: Each memory cell is associated with units of unique identifier, which allowing the CPU to locate and accesses data stored at a specific location.
  7. register: This is a special high-speed storage cell that holds the operands of an arithmetic operation and that, when the operation is complete, holds its result.
  8. fetch: This is process to retrieving or get data from memory and passing them to the control unit for decoding and execution.
  9. store: A process of saving data information in a storage or memory for future use.
  10. memory access time: This defined the time it takes for the computer's processors to carry out either a fetch or a store operation.
  11. DASD: A direct access storage devices is refers to a mass storage devices in which every memory cell has a unique address, and it allows data to be accesses directly by its address, rather than sequentially.
  12. Arithmetic and Logic Unit (ALU): This is the subsystem that performs such mathematical and logical operations as addition, subtraction, and comparison for equality.
  13. Control Unit: This is a component of the computer's central processing unit, which fetch from memory the next instruction to be executed, decode it, and execute it by issuing the appropriate command to the ALU, memory, or I/O controllers.

Chapter 6:
Four things the interface between the computer and the hardware does:
  1. Hides from the user the messy details of the underlying hardware
  2. Presents information about what is happening in a way that does not require in-depth knowledge of the internal structure of the system
  3. Allows easy user access to the resources available on this computer
  4. Prevents accidental or intentional damage to hardware, programs, and data
Responsibilities of systems software:
  1. Hides the complex and unimportant (to the user) details of the internal structure of the Von Neumann architecture
  2. Presents important information to the user in a way that is easy to understand
  3. Allows the user to access machine resources in a simple and efficient way
  4. Provides a secure and safe environment in which to operate
Terms
  1. Systems software: Is a collection of computer programs that manage the resources of a computer and facilitates access to those resources. This contrasts with application software that allows a user to address some specialized task of interest to that user, for example, write a document, create an image, browse the web, or solve a system of equations.
  2. user interface: A modern operating systems with a graphical user interface (GUI) that gives the user an intuitive visual overview as well as graphical control of the capabilities and services of the computer. Control of the GUI is typically done with keystrokes, mouse clicks, finger taps, voice activation, or biometric scans such as fingerprints.
  3. language services: This is called assemblers, compilers, and interpreters, allow you to write programs in a high-level, user-oriented language rather than the machine language and to execute these programs easily and efficiently. They often include components such as text editors and debuggers.
  4. memory managers: These programs allocate memory space for programs and data and retrieve this memory space when it is no longer needed.
  5. i/o managers: These software packages allow you to easily and efficiently use the many different types of input and output devices that exist on a modern computer system.
  6. scheduler: This system program keeps a list of programs ready to run on the processor, and it selects the one that will execute next. The scheduler allows you to have several different programs active at a single time, for instance, surfing the web while you are waiting for a file to finish printing.
  7. assembly language: These were termed second-generation languages to distinguish them from machine languages, which were viewed as first-generation languages.
Functions of an Operating system
  1. User interface management (a receptionist)
  2. Control of access to the system and to data files (a security guard)
  3. Program scheduling and activation (a dispatcher)
  4. Efficient resource allocation (an efficiency expert)
  5. Deadlock detection and error detection (a traffic officer)
