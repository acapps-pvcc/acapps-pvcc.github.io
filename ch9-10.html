Chapter 9: Introduction to High Level Languages
List four disadvantages of using assembly language:
  1. The programmer must “manually” manage the movement of data items between and among memory locations and registers (although such data items can be assigned mnemonic names).
  2. The programmer must take a microscopic view of a task, breaking it down into tiny subtasks—ADD, COMPARE, INCREMENT—that direct what is going on in individual memory locations and registers.
  3. An assembly language program is machine specific.
  4. Statements are not natural-language-like (although operations are given mnemonic code words as an improvement over a string of bits).
List four expectations programmer have for writing programs in a high-level language ("third generation" language):
  1. The programmer need not manage the details of the movement of data items within memory or pay any attention to exactly where those items are stored.
  2. The programmer can take a macroscopic view of tasks, thinking at a higher level of problem solving (add B and C, and call the result A). The “primitive operations” used as building blocks in algorithm construction.
  3. Programs are portable rather than machine specific.
  4. Programming statements are closer to natural language and use standard mathematical notation.
Briefly describe or define:
  1. object code: This refers to binary machine language instructions produced by a compiler or assembler after translating a program's source code or low-level assembly language. It is the machine-readable format that a computer can execute directly.
  2. source code: This refers to a set of human-readable instructions written by programmers in a high-level language. It typically needs to be translated into machine language by a compiler or interpreter before it can be executed by the computer.
  3. procedural language: A type of imperative programming language in which the program consists of sequences of statements that tell the computer what to do and how to do it. These instructions are executed in a step-by-step approach and typically manipulate data items to solve specific problems.
  4. Integrated Development Environment (IDE): An IDE is software that provides a collection of software development tools, such as a text editor to create a program; a file system to store programs; a language translator to translate the program to machine language; and a debugger to help locate the errors. It allows programmers to perform a number of tasks within the shell of a single application program, rather than having to use a separate program for each task.
  5. software development life cycle: The overall sequence of steps needed to complete a large-scale software project. These phases include the total time spent by programmers and designers on problem specification, program design, implementation, review, modification, debugging, improving the original code and writing final documentation.
  6. waterfall model of software development: It is defined as a sequential process that is one step must be completed before moving on to the next. Like a series of water falls moving a river down and down through various plateaus. In the software development process, these plateaus would be the feasibility study, problem specification, program design.
  7. agile model of software development: It is software development approach emphasizing flexibility ability to move with quick easy grace; resourceful and adaptable character, nimbleness, and the ability to adjust to changes. It involves close collaboration with the development team and the customer to ensure that the product meets their needs and requirements.
Describe each of these components of the software development life cycle:
  1. feasibility study: This evaluates a proposed project and compares the costs and benefits of various solutions. One choice might be to buy a new computer system for this project. In addition to the costs of the machine itself, there may be costs for peripherals such as 3-D printers or telecommunications links. The costs of software equipment maintenance; salary for developers or consultants, technical support people, and data entry clerks, and as must the costs incurred in training new users on the system.
  2. problem specification: It involves developing a clear, concise, and unambiguous statement of the exact problem the software is to solve. the original problem statement used in the feasibility study is written in a natural language, such as English, it may be unclear, incomplete, or even internally contradictory. This rough initial problem statement must be transformed into a complete, accurate, and consistent problem specification that describes the actions this software should take for every possible input it may encounter. Additionally, this phase, the software developers and their “customers”—those who are commissioning the software and will be its eventual users—must resolve each and every inconsistency, ambiguity, and gap.
  3. program design: It is the time to plan how it is to be done.Tasks are broken down into subtasks, which are broken down into sub-subtasks, until each piece is small enough to code comfortably. Some of this design may be documented graphically, through structure charts or through class diagrams that give the properties and functions of each class.
  4. algorithm selection: This step is the process of examining two different algorithms; a searching—sequential search and binary search which lets the programmer to determine which is more suitable for this particular task and which is more efficient.
  5. coding: It is the process of converting the detailed designs into computer code. If the design has been carefully developed, this should be a relatively routine job. Perhaps reusable code can be pulled from a program library, or a useful class can be employed.
  6. debugging: Is the process of locating and correcting program errors, and it can be a slow and expensive operation that requires as much effort as writing the program in the first place. Errors can occur because a program statement fails to follow the correct rules of syntax, which makes the statement unrecognizable by the compiler and results in an error.
  7. testing: This approach is to design a special set of test cases, called a test suite, and run the program using this special test data. Test data that is carefully chosen to exercise all the different logic paths through a program can help uncover errors.
  8. verification: Is the process used to prove that if the input data to a program satisfies certain conditions, then, after the program has been run on this data, the output data satisfies certain other conditions. This is a formal proof, much like the proof of a mathematical theorem that condition B follows from condition A.
  9. documentation: It refers to all of the written and online material that makes a program understandable. This includes internal documentation, which is part of the program code itself. Good internal documentation consists of choosing meaningful names for program identifiers, using plenty of comments to explain the code, and separating the program into short modules, each of which does one specific subtask. External documentation consists of any materials assembled to clarify the program’s design and implementation.

Questions & Answers:

  • Q: Why does the textbook refer to "the Tower of Babel" regarding computer languages?
    A: The textbook uses the ”Tower of Babel“ analogy because computer languages are numerous and complex; each language has different characteristics and unique expressions, which can be confusing. This is similar to the biblical story of Babel, which illustrates how the diversity of human languages has made communication complicated, just as the challenge comes from using and understanding different programming languages to write programs.
  • Q: Why isn't there just one computer programming language?
    A: There isn't just one programming language because each language was designed to meet the specific needs of one particular area of application. Consequently, one language might be better suited than others for writing certain kinds of programs. The situation is somewhat analogous to the automobile market. The basic automotive needs of the country probably could be served by a single car model and a single truck model.
  • Q: Why didn't FORTRAN use the less than ( ‹ ) symbol when comparing two numbers?
    A: FORTRAN doesn't use less the than (<) symbol when comparing two numbers because the keypunches that were used to create the punched cards on which early Fortran programs were submitted to the computer had no such symbols.
  • Q: What was COBOL designed for?
    A: COBOL was designed to serve business needs such as managing inventories and payrolls. In such applications, summary reports are important output products. Much of the processing in the business world concerns updating “master files” with changes from “transaction files.”
  • Q: What was the language, C, designed for?
    A: The C language was designed for systems programming, in particular for writing the operating system UNIX. UNIX had been developed at Bell Labs a short time before and was originally written primarily in assembly language.
  • Q: What is the difference between C and C++?
    A: The difference between C and C++ is that C does not support object oriented programming, while C++ is an object oriented programming language that adds many new features; giving it more sophistication and cleaner ways to do certain tasks.
  • Q: Who or what is the language, Ada, named for and why?
    A:The programming language Ada was named after Ada Augusta Byron Lovelace. Lady Ada Lovelace is regarded as the world’s first programmer based on her correspondence with Charles Babbage and her published notes on his work with the Analytical Engine.
  • Q: What is a Java "applet"?
    A: Java "applet" is a small application that designed to run from webpages.
  • Q: How much does Python cost to download, install, and use?
    A:Python is a open source programming language that is free to download, and is freely available and can be used, distributed, or modified by anyone.
  • Q: What is Microsoft.NET?
    A: Microsoft.NET is essentially a giant collection of tools for software development. It was designed so that traditional text-based applications, GUI applications, and web-based programs can all be built with equal ease. The.NET Framework provides a whole library of classes for building GUIs with menus, buttons, text boxes, and more.
  • Q: What is SQL and what is it used for?
    A: SQL stands for Structured Query Language and it a specialized language. It is designed to be used with databases, which are collections of related facts and information. It allows user to be able to add new data, create and retrieve data already stored.
  • Q: What is R and what is it used for?
    A: R is a specialized programming language designed for statistical computing and graphics. It can compute the maximum value, minimum value, mean, median, standard deviation, and other more sophisticated statistical functions with data set. unlike a spread sheet, in R program it allows multiple data sets to be run efficiently and repeatedly.
  • Q: What is Go (or golang) and what is it used for?
    A: Go is a programming language developed at Google. It was designed to combine the ease of programming of an interpreted, dynamically typed language with the efficiency and safety of a statically typed, compiled language. Go is used for systems programming, programming for multicore machines.
  • Q: What is Swift and what is it used for?
    A: Swift is a modern programming language developed by Apple. It is used for building applications for Apple's platforms, including iOS (iPhone/iPad), macOS, watchOS, and tvOS. In other words, Swift powers many of the apps across Apple devices.


Briefly describe each of the following alternative programming paradigms
  1. functional programming: It views every task in terms of (surprise!) functions. Unlike the more general usage of the word function in some procedural programming languages.
  2. logic programming: This type of programming go a step further toward not specifying exactly how a task is to be done. In logic programming, various facts are asserted to be true, and on the basis of these facts, a logic program can infer or deduce other facts. When a query is posed to the program, it begins with the storehouse of facts and attempts to apply logical deductions, in as efficient a manner as possible, to answer the query.
  3. parallel programming: This is a catchall term for a variety of approaches to computing architectures and algorithm design.

In C#,the statement used to get input from the user is Console.ReadLine(), which reads the value entered by the user, and Console.Write is used to display text to the user. In Java, the statement used to get input from the user is inp.nextInt(), which reads input value from the user. In Python, the statement used to get input from the user is input(), which reads a value entered by the user.

The statement used to display output to the user include: Ada is TEXT_IO.PUT, in C++ it used cout << to display output to the user, C# used Console.Write statement to display output, Java used System.out.println() to print out output to the user, Python to print out output to the user is print() statement.
